# CS-360-8-3-Journal
For this project, I have chosen to develop the Event-Tracking app. The overarching purpose of the app is to give users a central point for managing, planning, and organizing gatherings of any kind. The major components that support the application's goals include a login/create account screen, a Pinterest-style grid screen, a notification system, and databases one for login information and one for a catalog of event details. The most important component of the app is event management, where users can add new events, remove past or canceled events, and edit details such as the time and location.
At a high level, the app focuses on a visually-driven approach inspired by Pinterest, with layouts that include color coding and icons to distinguish different events. Two apps similar to mine are Google Calendar, which shows events in a timeline or calendar view to help people organize work and personal schedules, and Eventbrite, which helps users find and track events they plan to attend. My Event-Tracking App differs by using a Pinterest-style grid, showing events as cards that are easy to scan and visually engaging.
The app targets three main types of users. Busy professionals need to track meetings, deadlines, and social events. Event planners manage multiple events and require a simple way to see and modify details. Social organizers attend or host events frequently and want to stay up-to-date. This app helps users by providing a simple, visual way to see all events at once, quickly add events, or check details. The Pinterest-style layout makes scrolling fun, and notifications ensure important events are not missed.
To support these user needs, the app includes several key screens and features: a Home/Dashboard displaying upcoming events at a glance, Event Creation and Detail screens for adding or editing events, Settings/Profile for personalization, and Notifications/Reminders to keep users on track. The UI design prioritizes clarity, simplicity, and accessibility, with consistent navigation and uncluttered layouts, ensuring users can interact efficiently and naturally.
My coding approach focused on modular development, creating reusable components for event cards, input forms, and navigation elements. I followed an incremental development strategy, implementing one feature at a time, and used GitHub for version control to maintain an organized workflow. These strategies made debugging easier and ensured maintainable, scalable code—skills that will be valuable for future projects.
Testing was conducted through both manual interaction and debugging tools to verify functionality, such as adding events, editing details, and triggering notifications. This process revealed minor issues, like event data not saving correctly or display inconsistencies, which were promptly resolved. Testing ensures a reliable app experience and validates that the UI and features work as intended.
Throughout development, challenges such as handling dynamic event data and implementing timely notifications required innovative solutions, including designing efficient data structures for events and creating a notification system that adapts to user preferences. The component where I demonstrated my skills most effectively was the event dashboard, which dynamically displays upcoming events in the Pinterest-style grid, showcasing both technical competence and attention to user-centered design principles.
Overall, this project demonstrates my ability to develop a functional, visually appealing, and user-friendly mobile app while applying best practices in coding, UI design, and testing, all tailored to meet the diverse needs of event organizers and attendees.
